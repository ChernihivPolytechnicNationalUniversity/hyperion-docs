## Step-by-Step Instructions

### Step 1: Clone Repository

**Objective**: Get the source code on your local machine

**Instructions**:

1. Clone the repository:
   ```bash
   git clone [repository-url]
   cd digital-university
   ```

2. Verify the clone:
   ```bash
   git status
   ```

**Expected Result**: You should see "On branch main" with a clean working directory

**Troubleshooting**:
- **Issue**: Permission denied
  **Solution**: Ensure SSH keys are configured or use HTTPS URL

### Step 2: Install Dependencies

**Objective**: Install project dependencies

**Instructions**:

1. Install packages:
   ```bash
   npm install
   # or
   pnpm install
   ```

2. Verify installation:
   ```bash
   npm list --depth=0
   ```

**Expected Result**: All dependencies installed without errors

### Step 3: Configure Environment

**Objective**: Set up environment variables

**Instructions**:

1. Copy environment template:
   ```bash
   cp .env.example .env.local
   ```

2. Edit `.env.local` with your values:
   ```bash
   DATABASE_URL=postgresql://localhost:5432/du_dev
   API_KEY=your-api-key-here
   ```

**Expected Result**: Environment file configured with valid values

### Step 4: Start Development Server

**Objective**: Run the application locally

**Instructions**:

1. Start the server:
   ```bash
   npm run dev
   ```

2. Open browser to http://localhost:3000

**Expected Result**: Application loads successfully

## Best Practices

- Keep dependencies up to date
- Use environment variables for configuration
- Follow project code style

## Common Pitfalls

### Pitfall 1: Port Already in Use
**Problem**: Server fails to start with EADDRINUSE error
**Solution**: Kill the process on port 3000 or use a different port
